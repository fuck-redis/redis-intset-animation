### 核心算法

#### 二分查找

```typescript
binarySearch(value: number): SearchResult {
  let left = 0;
  let right = length - 1;
  
  while (left <= right) {
    const mid = Math.floor((left + right) / 2);
    if (contents[mid] === value) return { found: true, index: mid };
    if (contents[mid] < value) left = mid + 1;
    else right = mid - 1;
  }
  
  return { found: false, index: left };
}
```

#### 编码升级

```typescript
insert(value: number): IntSetState {
  // 检查是否需要升级编码
  if (!fitsEncoding(value, currentEncoding)) {
    encoding = getNextEncoding(currentEncoding);
    // 迁移现有数据到新编码
    migrateToNewEncoding(encoding);
  }
  
  // 插入新值
  const position = binarySearch(value).index;
  contents.splice(position, 0, value);
  
  return newState;
}
```

## 🎓 教育价值

### 学习目标

通过本系统，你将了解：

1. **IntSet数据结构**
   - 紧凑的内存存储方式
   - 自动编码升级机制
   - 有序集合的维护策略

2. **算法复杂度**
   - 二分查找：O(log n)
   - 插入操作：O(n)（需要移动元素）
   - 查找效率与数组大小的关系

3. **内存管理**
   - 不同编码下的内存占用
   - 编码升级的代价
   - 空间效率与值范围的权衡

### 使用场景

IntSet在Redis中用于存储**整数类型的Set**，当满足以下条件时启用：
- 所有元素都是整数
- 元素数量不超过配置阈值（默认512）

## 📖 使用示例

### 基本操作流程

1. **创建集合**：系统默认创建包含[1, 3, 5]的IntSet
2. **插入元素**：输入整数，点击执行，观察插入动画
3. **触发升级**：插入40000，观察INT16→INT32的升级过程
4. **查找元素**：输入值进行查找，查看二分查找路径
5. **删除元素**：移除指定元素，观察后续元素的移动

### 测试场景

#### 场景1：正常插入
```
初始状态：[1, 3, 5] (INT16)
插入 4
结果：[1, 3, 4, 5] (INT16)
```

#### 场景2：触发升级
```
初始状态：[100, 200, 300] (INT16)
插入 40000
结果：[100, 200, 300, 40000] (INT32)
内存：6字节 → 16字节
```

#### 场景3：批量插入
```
输入：10, 20, 30, 40, 50
结果：自动排序并去重，一次性展示
```
